# 基座模型与 MDL 描述符优化

## 基座模型的设计理念

对于酶设计领域，一个理想的基座模型（foundation model）应该：

1. **理解底物**：从分子结构中提取有化学意义的表示
2. **理解酶**：从序列/结构中提取活性位点的关键特征
3. **理解条件**：将温度、溶剂、pH 等连续/离散条件编码为可操作的潜变量
4. **理解三者交互**：底物×酶×条件的联合嵌入空间

### 预训练策略

基座模型的核心价值在于：大规模预训练捕获通用化学知识 → 少量领域数据微调适配特定酶体系。

预训练数据可以来自：
- 公共反应数据库（USPTO、Pistachio、Reaxys）
- 酶反应数据库（BRENDA、UniProt 的功能注释）
- 蛋白质结构数据库（PDB 中酶-底物共晶结构）
- 文献文本（利用 NLP 从方法部分提取反应条件）

详见 [ml4e 项目](https://github.com/nietzsche/ml4e) 中关于 PDB 蛋白质-小分子结构的分析。

### 微调策略

对于特定的酶体系（如 DERA-MA），微调数据的构建是关键挑战：

- **正样本**：成功的反应条件和对应的产率
- **负样本**：失败的反应（产率 < 阈值）——同样有价值，但往往不被系统记录
- **数据量**：可能只有几十到几百个数据点

这就需要结合[迁移学习](transfer-learning-enzyme.md)的方法——利用机理相似性扩张有效数据量。

## MDL 方法优化描述符

### 什么是 MDL？

最小描述长度（Minimum Description Length）原则提供了一个信息论框架来评估描述符的有效性：**最优的描述符集合是在给定精度下用最少的比特数描述数据的那个**。

在化学描述符的上下文中：
- 每个描述符都有"成本"——增加模型的复杂度
- 描述符组合有"收益"——提高预测精度
- MDL 自动平衡成本与收益，选出最简约有效的描述符子集

### 方法框架

1. **数据集**：使用 freesolv（水合自由能）、毒性数据集等标准基准
   - 这些数据集有明确的物理化学含义，便于验证描述符的物理可解释性
2. **Baseline 选择**：
   - XGBoost（选一个"没那么好"的模型——保留改进空间，同时足够强以排除弱描述符）
   - 随机森林（RF）——对描述符共线性不敏感
   - 树方法家族——天然适合处理混合类型描述符
3. **搜索策略**：在描述符空间中搜索 MDL 最优子集
4. **验证**：MDL 选出的描述符是否在物理化学上有意义？是否与已知的物理有机化学原理一致？

## 描述符的物理基础

描述符选择不仅是工程问题，更是科学问题。核心选择：

- **电性描述符**：Hammett σ、NBO 电荷、HOMO/LUMO 能级、静电势
- **空阻描述符**：Taft Es、Sterimol 参数、%Buried Volume
- **拓扑描述符**：分子连接性指数、ECFP 指纹（牺牲部分可解释性）

JC 关于 Sigman 描述符的设计工作提供了重要参考——如何设计既有物理意义又有预测力的描述符。详见[学习路线图](learning-roadmap.md)中的相关内容。

## 从描述符到预测：SMC 与 QSAR

**构效关系（QSAR）是物理有机化学的核心**。但一个深刻的洞察是（SMC 的观点）：即使在分子层面完全理解机理，要设计新的分子和反应条件仍然极其困难。这是因为：

1. 机理理解通常是定性的（知道什么因素重要），而设计需要定量（知道多重要）
2. 多因素的协同/拮抗效应难以从单个因素的知识中预测
3. 活性位点的动态构象空间远大于静态晶体结构所呈现的

因此，基座模型和描述符优化不是为了替代机理理解，而是为了在机理理解提供的框架内，完成从定性到定量的转换。
